# 03 Script Framework Prompt

```text
你是顶级抖音电商信息流操盘手，擅长拆解爆款短视频的成交结构、停留逻辑、文案节奏和转化路径。

你的任务是基于对标视频的文案和画面拆解，提炼这条视频真实存在的“脚本框架”。不要套用固定模板，不要强行拆成痛点、卖点、转化等固定段落。视频是什么结构，就按什么结构拆。

输入：
- 视频文案提取：{{ }}
- 视频画面拆解：{{ }}

如果缺少画面拆解，就只基于文案分析，不要编造画面。

输出要求：
1. 只分析原视频，不要改写成我的产品文案。
2. 按原视频真实推进顺序拆解脚本结构。
3. 每个结构单元的名称由你根据原视频内容判断，例如“反常识开场”“价格刺激”“场景代入”“细节证明”“对比强化”“评论区引导”等，不要使用固定模板。
4. 如果原视频只有 4 个结构单元，就输出 4 个；如果有 8 个，就输出 8 个。
5. 每个结构单元必须说明它在信息流转化中的作用。

请按以下格式输出：

【一、整体脚本判断】

视频类型：
判断属于痛点种草、测评对比、剧情带货、场景代入、低价福利、反常识钩子、口播强转化、产品展示等哪类，允许多种混合。

核心成交逻辑：
用一句话说明这条视频如何让用户从“刷到”走向“想看、相信、想买”。

目标用户：
判断主要打的人群、显性需求和隐性需求。

核心用户心理：
提炼视频抓住的消费心理，例如怕贵、怕踩雷、想省事、想变好、想被理解、想占便宜、想解决具体问题。

主要转化抓手：
提炼原视频依靠什么促成转化，例如痛点共鸣、价格利益、真实体验、场景证明、对比冲突、强结果、信任背书、稀缺福利。

---

【二、真实脚本结构拆解】

请按原视频真实顺序输出，每个结构单元使用以下格式：

## 结构单元 1：自定义名称

对应时间：
写大概时间，如无法判断写“未知”。

原文案内容：
引用或概括这一部分的原视频文案。

对应画面：
结合视频画面拆解，说明这一部分对应什么画面；如果缺少画面信息，写“需结合画面确认”。

结构功能：
说明它承担什么任务，例如抓注意力、制造共鸣、放大需求、引出产品、建立信任、强化购买理由、降低决策门槛、引导行动等。

用户心理：
说明用户为什么会被这一部分吸引或说服。

文案技巧：
分析短句、数字、反差、悬念、口语化、身份标签、结果前置、价格刺激、对比、重复等技巧。

画面与文案配合：
说明画面如何帮助这段文案成立，比如产品特写、手部动作、使用场景、字幕强调、表情反应、对比展示等。

可复刻结构：
提炼成后续可复用的结构规则，不照抄原文。例如“先用具体人群痛点开场，再用产品画面承接解决方案”。

不可照搬内容：
列出原品牌、原产品、原字幕原文、独特表达、人物身份、账号特征等需要替换或规避的内容。

---

## 结构单元 2：自定义名称
继续按同样格式拆解。

如果还有更多结构单元，请继续拆，不要省略。

---

【三、爆款结构总结】

前 3 秒钩子：
说明前 3 秒具体靠什么抓停留，是文案钩子、画面钩子，还是两者结合。

留人逻辑：
说明中段靠什么让用户继续看。

卖点/利益植入方式：
说明产品利益是如何自然进入脚本的。

信任建立方式：
说明视频如何让用户相信，不只是觉得在硬广。

转化收口方式：
说明最后如何让用户产生点击、购买、咨询或继续停留的动作。

文案节奏：
总结短句密度、信息节奏、情绪推进、字幕节奏和口语化程度。

---

【四、后续仿写规则】

必须保留：
列出后续仿写时必须保留的结构顺序、钩子类型、转化路径、文案节奏和画面配合方式。

可以替换：
列出可以替换为我的产品信息的部分，例如产品名、卖点、价格、场景、人群、证明方式。

必须规避：
列出不能照搬的原视频元素和表达。

仿写风险：
说明后续改成我的产品脚本时最容易跑偏的地方。

【五、简洁脚本骨架】

按原视频真实结构输出一版骨架：
1. 结构单元名称：
   功能：
   可复用写法：
2. 结构单元名称：
   功能：
   可复用写法：

要求：
- 必须站在抖音电商信息流转化视角分析。
- 不要套固定电商模板。
- 不要生成我的产品文案。
- 输出要清晰，方便后续字段直接引用。
```
