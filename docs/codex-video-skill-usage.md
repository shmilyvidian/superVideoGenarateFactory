# Codex 视频生成 Skill 使用说明

这份文档用于说明如何跟 Codex 沟通，让 `xborder-video-replication` skill 帮你完成带货短视频生成。

当前 skill 支持三条链路：

1. 复刻链路
2. 强 Hook 九宫格生成出片链路
3. 九宫格成片直投链路

图片和视频生成都经 X-Border 中转调用，用户侧零 key，无需配置任何密钥；Provider key 由 Worker 服务端持有，不要把任何密钥写进聊天、文档、prompt、代码或仓库。

Seedance 出片默认保留真实环境音和动作音效。只有你明确要求静音时，才使用静音模式。

---

## 一、推荐使用方式

每次开始前，先明确告诉 Codex 走哪条链路。

通用开头：

```text
$xborder-video-replication

走【链路名称】。
```

如果要让 Codex 最后自动调用 Seedance 2.0，可以补一句：

```text
最后调用 Seedance 2.0 出视频。
```

如果只想先看脚本或提示词，可以说：

```text
先不要出视频，只生成脚本和提示词给我确认。
```

---

## 二、链路一：复刻链路

适合场景：

你有一个对标视频，希望 Codex 拆解它的镜头、文案、节奏和画面结构，再替换成你的产品。

流程：

```text
对标视频
→ Codex 抽帧拆解镜头
→ 提取口播/字幕/前3秒 Hook
→ 生成复刻蓝图
→ 仿写文案
→ 生成新产品分镜图
→ 生成 Seedance 2.0 视频提示词
→ 调用 Seedance 2.0 出视频
```

推荐对话模板：

```text
$xborder-video-replication

走复刻链路。

我会上传一个对标视频和我的产品图。
请先拆解对标视频的镜头、动作、文案和节奏，再替换成我的产品。
生成分镜图和 Seedance 2.0 视频提示词，最后调用 Seedance 2.0 出视频。

产品信息：
产品名称：
核心卖点：
目标人群：
注意事项：
```

适合补充的要求：

```text
前3秒 Hook 一定要保留对标视频的停滑逻辑。
产品外观必须以我的产品图为准，不要继承对标视频里的产品。
如果产品 logo 或包装细节应该可见，必须保留在正确位置。
```

## 三、链路二：强 Hook 九宫格生成出片链路

适合场景：

你没有对标视频，想让 Codex 从商品信息开始，自动生成一条带货视频。

这是目前重点链路。

流程：

```text
商品信息 / 卖点 / 目标人群 / 产品图
→ Codex 读取 prompt_image.md
→ 生成强 Hook 中文 9 宫格分镜脚本
→ 等你确认
→ 生成 9 段 image2 生图提示词
→ 调用 image2 生成 9 帧分镜图
→ 根据 9 帧图 + 9 段提示词生成 Seedance 2.0 视频提示词
→ 调用 Seedance 2.0 出视频
```

这条链路的重点：

- 前 3 格必须有强 Hook。
- 第 1 格必须有停滑点。
- 第 2 格继续放大痛点或制造好奇。
- 第 3 格必须出现产品、方案或关键转折。
- 如果前 3 格太普通，Codex 必须重写前 3 格。

推荐对话模板：

```text
$xborder-video-replication

走强 Hook 九宫格生成出片链路。

商品名称：
核心卖点：
目标人群：
使用场景：
视频语言：
视频时长：
视觉风格：
声音模式：ambient / music / voiceover / full / silent

我会上传产品图。
请先根据 prompt_image.md 生成 9 宫格中文分镜脚本。
前三格一定要夸张一点，要有停滑点，不要普通产品展示。
生成中文脚本后先停下来等我确认。
```

示例：

```text
$xborder-video-replication

走强 Hook 九宫格生成出片链路。

商品名称：苹果果茶
核心卖点：鲜切苹果、茶底清爽、奶盖顺滑、夏天解腻
目标人群：18-30 岁喜欢奶茶果茶的女生
使用场景：下午茶、逛街、办公室提神
视频语言：中文
视频时长：9 秒
视觉风格：明亮、清爽、夏日感、强商业广告质感
声音模式：full，真实环境音 + 动作音效 + 轻快背景音乐 + 女生口播

我会上传产品图。
请先生成 9 宫格中文分镜脚本，前三格一定要有强 Hook。
生成后先停下来等我确认。
```

确认脚本后，可以继续说：

```text
确认，继续生成 9 段 image2 生图提示词。
```

如果 Hook 不够强，可以说：

```text
第 1 格不够抓人，重新加强前 3 格 Hook。
```

生成 image2 JSON 后，可以说：

```text
确认，调用 image2 生成 9 帧分镜图。
```

分镜图确认后，可以说：

```text
确认，用这 9 帧分镜图和 9 段提示词调用 Seedance 2.0 出视频。
```

## 四、链路三：九宫格成片直投链路

适合场景：

你已经有 9 帧分镜图和 9 段提示词，这些提示词通常是根据 `prompt_image.md` 生成的。

Codex 不再重新写脚本，也不再调用 image2，只负责把这套素材整理成 Seedance 2.0 视频提示词，并调用 Seedance 2.0 出视频。

流程：

```text
你上传 9 帧分镜图 + 9 段提示词
→ Codex 整理 Seedance 2.0 视频提示词
→ 调用 Seedance 2.0 出视频
```

推荐对话模板：

```text
$xborder-video-replication

走九宫格成片直投链路。

我会上传 9 帧分镜图和 9 段提示词。
这些提示词是根据 prompt_image.md 生成的。
不要重新生成脚本，不要调用 image2。
请直接根据这 9 帧图和 9 段提示词整理 Seedance 2.0 视频提示词，并调用 Seedance 2.0 出视频。
```

如果你还想控制视频时长，可以加：

```text
视频总时长控制在 9 秒左右，每个分镜约 1 秒。
```

或：

```text
视频总时长控制在 15 秒以内，根据 9 个分镜自然分配时间。
```

## 五、声音配置

Seedance 生成视频时可以直接生成声音。建议不要默认静音，因为很多真实环境音和动作音效后期很难自然补出来。

可选声音模式：

```text
audio_mode=ambient
默认模式。保留真实环境音和动作音效，比如倒水声、开盖声、冰块声、包装摩擦声、脚步声、产品接触声。不要人声，不要背景音乐。
```

```text
audio_mode=music
环境音 + 动作音效 + 背景音乐。不要人声口播。
```

```text
audio_mode=voiceover
环境音 + 动作音效 + 人声口播/旁白。背景音乐不生成或只保留极轻铺底。
```

```text
audio_mode=full
环境音 + 动作音效 + 背景音乐 + 人声口播全部允许。
```

```text
audio_mode=silent
静音模式。只有明确想要无声音视频时使用。
```

常用说法：

```text
调用 Seedance 2.0 时使用 audio_mode=ambient，只要真实环境音和动作音效，不要人声，不要背景音乐。
```

```text
调用 Seedance 2.0 时使用 audio_mode=full，要有真实环境音、动作音效、轻快背景音乐和女生口播，不要字幕。
```

```text
调用 Seedance 2.0 时使用 audio_mode=music，保留动作音效，加轻快背景音乐，不要口播。
```

如果你有参考音乐 URL，可以补充：

```text
我会提供参考音乐 URL，请作为 reference_audio 使用。
```

## 六、常用确认语

确认脚本：

```text
确认这个中文分镜脚本，继续生成 image2 提示词。
```

要求加强 Hook：

```text
前三格不够抓人，请重写前 3 格 Hook，要更夸张、更有停滑点，但不要虚假宣传。
```

确认生成图片：

```text
确认，调用 image2 生成 9 帧分镜图。
```

确认出视频：

```text
确认，用这些分镜图和提示词调用 Seedance 2.0 出视频。
```

只要提示词，不出视频：

```text
先不要调用 Seedance，只输出最终视频生成提示词。
```

重新出视频：

```text
这版视频不够好，请保留当前 9 帧分镜图，重新优化 Seedance 提示词再出一版。
```

---

## 七、注意事项

1. 图片和视频都经 X-Border 中转调用，用户侧零 key，无需配置或保管任何密钥。
2. Provider key 由 Worker 服务端持有，永远不会下发到客户端；不要在聊天、文档、代码或仓库里粘贴任何密钥。
3. 复刻链路里的对标视频默认是给 Codex 拆解用，不是直接作为 Seedance 的 `reference_video`。
4. 当前脚本已经验证过基础 Seedance 出片链路：图片参考 + 文本 prompt 可以创建任务、轮询并下载 MP4。
5. 当前已经支持 `reference_audio` URL；如果未来要支持官方示例里的 `reference_video`，还需要继续扩展 `seedance_submit.py`。
